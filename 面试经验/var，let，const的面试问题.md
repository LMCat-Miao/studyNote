## 问题1：

### const 定义的变量可以修改吗？

回答：

> const 声明的变量不能重新赋值，但是如果变量保存的是对象或数组，内部的数据仍然可以修改，因为 const 限制的是引用地址，而不是对象内容。

------

## 问题2：

### 为什么 const obj={} 可以修改 obj.name？

回答：

> 因为 obj 保存的是对象的引用地址，修改 obj.name 只是修改地址指向对象内部的数据，并没有改变 obj 指向的地址。

------

## 问题3：

### const 和 Object.freeze 有什么区别？

回答：

|          | const    | Object.freeze |
| -------- | -------- | ------------- |
| 限制     | 变量地址 | 对象内容      |
| 属性修改 | 允许     | 禁止          |
| 重新赋值 | 禁止     | 不影响        |
| 作用层级 | 变量     | 对象          |

## 问题4：

### var、let、const区别？

标准答案：

> var是ES5声明变量方式，存在变量提升，没有块级作用域，可以重复声明。let和const是ES6新增，都具有块级作用域，不允许重复声明。let可以重新赋值，而const声明后不能重新赋值。开发中优先使用const，需要修改时使用let。

------

## 问题5：

### 什么是变量提升？

回答：

> JavaScript执行代码前，会先进行变量和函数声明的提升。var声明的变量会被提升，并初始化为undefined，所以可以在声明前访问。

------

## 问题6：

### let有没有变量提升？

高级回答：

> let和const存在提升过程，但是不会初始化，在声明之前访问会进入暂时性死区TDZ，因此不能访问。

------

## 问题7：

### const对象可以修改吗？

回答：

> 可以修改对象内部属性，因为const限制的是变量引用地址不能改变，而不是对象内容不可变。

------

## 问题8：

### 为什么不用var？

回答：

> var存在函数作用域、变量提升、重复声明等问题，容易造成代码维护困难。let和const提供块级作用域，可以减少变量污染。

## var、let、const 和作用域有什么区别？

你可以这样回答：

> var 是函数作用域变量，它不会产生块级作用域，在 if、for 等代码块中声明后外部仍然可以访问。let 和 const 是 ES6 引入的块级作用域变量，只在当前 {} 内有效，可以避免变量污染。const 和 let 的作用域规则一样，区别在于 const 不允许重新赋值。
