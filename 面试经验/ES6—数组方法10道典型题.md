# 题1：map转换接口数据

## 原数据

```
const users=[
{
id:1,
username:"Tom",
age:20
},
{
id:2,
username:"Jack",
age:18
}
]
```

需求：

转换成：

```
[
{
label:"Tom",
value:1
}
]
```

------

## 面试答案

```
const options = users.map(user=>{

    return {
        label:user.username,
        value:user.id
    }

})
```

------

## 执行过程

map会遍历每一个元素。

第一次：

```
user={
id:1,
username:"Tom"
}
```

执行：

```
return {
label:user.username,
value:user.id
}
```

得到：

```
{
label:"Tom",
value:1
}
```

第二次：

得到：

```
{
label:"Jack",
value:2
}
```

最后：

```
[
{
label:"Tom",
value:1
},
{
label:"Jack",
value:2
}
]
```

------

## 面试官追问：

### map会改变原数组吗？

回答：

不会。

因为map返回新的数组。

# 题2：map + 展开更新

需求：

修改：

```
id=2
```

用户：

```
active:true
```

------

答案：

```
const newUsers = users.map(user=>{

    if(user.id===2){

        return {
            ...user,
            active:true
        }

    }

    return user

})
```

------

## 面试官重点看什么？

这里考：

### 1. map生成新数组

外层：

```
map
```

保证：

```
newUsers !== users
```

------

### 2. 展开保证对象不被修改

错误：

```
user.active=true
```

问题：

直接修改旧对象。

正确：

```
{
...user,
active:true
}
```

创建新对象。

# 题8：reduce购物车总价 ⭐⭐⭐⭐⭐

## 数据

```
const cart=[

{
name:"iphone",
price:8000,
count:1
},

{
name:"ipad",
price:4000,
count:2
}

]
```

需求：

计算：

```
8000*1 + 4000*2
```

答案：

```
const total=cart.reduce((sum,item)=>{

    return sum + item.price * item.count

},0)
```

------

# 重点：reduce执行过程（一定掌握）

reduce有两个东西：

```
sum
item
```

sum：

累计结果。

item：

当前商品。

初始值：

```
0
```

------

## 第一次执行

reduce开始：

```
sum=0
```

当前：

```
item={
name:"iphone",
price:8000,
count:1
}
```

执行：

```
sum + price*count
```

也就是：

```
0+8000*1
```

结果：

```
8000
```

现在：

```
sum=8000
```

------

## 第二次执行

当前：

```
item={
name:"ipad",
price:4000,
count:2
}
```

执行：

```
8000+4000*2
```

等于：

```
16000
```

最终：

```
total=16000
```

------

## 面试官追问：

为什么不用for循环？

回答：

> reduce可以表达数组聚合逻辑，使代码更加声明式，并且适合统计、分组、转换结构。

# 题9：reduce数组转对象 ⭐⭐⭐⭐⭐

这是很多校招面试喜欢问的。

原数据：

```
[
{
id:101,
name:"Tom"
},
{
id:102,
name:"Jack"
}
]
```

目标：

```
{
101:{
id:101,
name:"Tom"
},

102:{
id:102,
name:"Jack"
}
}
```

------

答案：

```
const userMap=users.reduce((result,user)=>{


    result[user.id]=user;


    return result;


},{})
```

------

现在慢慢拆。

reduce初始值：

```
{}
```

也就是：

第一次：

```
result={}
```

------

## 第一次循环

当前：

```
user={
id:101,
name:"Tom"
}
```

执行：

```
result[user.id]=user
```

替换：

```
result[101]=user
```

结果：

```
{
101:{
id:101,
name:"Tom"
}
}
```

然后：

```
return result
```

交给下一次。

------

## 第二次循环

现在：

```
result={
101:{
...
}
}
```

当前：

```
user={
id:102,
name:"Jack"
}
```

执行：

```
result[102]=user
```

得到：

```
{
101:{
id:101,
name:"Tom"
},

102:{
id:102,
name:"Jack"
}
}
```

------

## 为什么return result？

这是reduce最容易错的地方。

如果：

```
return result
```

没有写：

下一轮：

```
result
```

会丢失。

------

## 工作场景

为什么要数组转对象？

因为查询效率。

数组：

找id=102：

```
users.find()
```

需要遍历。

对象：

```
userMap[102]
```

直接访问。

后台管理系统大量使用。

# 题10：综合业务题 ⭐⭐⭐⭐⭐

需求：

找：

1. 年龄 >18
2. role="admin"
3. active=true

然后：

只返回名字。

------

数据：

```
[
{
name:"Tom",
age:20,
role:"admin",
active:true
},

{
name:"Jack",
age:16,
role:"user",
active:true
},

{
name:"Lucy",
age:22,
role:"admin",
active:false
}
]
```

------

正确答案：

```
const result=users
.filter(user=>{

    return user.age>18
    &&
    user.role==="admin"
    &&
    user.active

})
.map(user=>{

    return user.name

})
```

------

# 分步骤分析

## 第一步 filter

目标：

筛选符合条件的人。

### Tom

年龄：

```
20>18 true
```

角色：

```
admin true
```

状态：

```
true
```

保留。

------

### Jack

年龄：

```
16>18 false
```

删除。

------

### Lucy

年龄：

```
22>18 true
```

角色：

```
admin true
```

状态：

```
false
```

删除。

------

filter之后：

```
[
{
name:"Tom",
age:20,
role:"admin",
active:true
}
]
```

------

## 第二步 map

现在只需要名字。

执行：

```
user.name
```

得到：

```
[
"Tom"
]
```

------

# 为什么不能直接map？

错误：

```
users.map(user=>{

if(condition){

return user.name

}

})
```

结果：

```
[
"Tom",
undefined,
undefined
]
```

因为：

map一定返回相同长度。

------

所以：

先filter：

减少数据。

再map：

转换数据。

这是企业代码经典模式：

```
filter
+
map
```

# 最后给你一个面试总结

## map

关键词：

> 转换

例：

```
users.map(user=>user.name)
```

------

## filter

关键词：

> 筛选

例：

```
users.filter(user=>user.age>18)
```

------

## find

关键词：

> 找一个

例：

```
users.find(user=>user.id===1)
```

------

## some

关键词：

> 有没有

例：

```
users.some(user=>user.role==="admin")
```

------

## every

关键词：

> 是否全部

例：

```
users.every(user=>user.age>=18)
```

------

## reduce

关键词：

> 汇总 / 转换结构

例：

```
reduce((sum,item)=>sum+item.price,0)
```
