## Q：NaN 是什么类型？

答案：

> NaN 是 Number 类型，表示一个无法表示的数字结果。

## null 和 undefined 区别？

回答：

> undefined 表示变量声明但没有赋值，通常由 JavaScript 自动产生；null 表示开发者主动设置为空，用于表示“没有对象”

## 问题1：

### 为什么 typeof NaN 是 number？

回答：

> NaN 是 JavaScript Number 类型中的特殊值，表示一个无法表示的数字结果，而不是表示它不是数字类型。

------

## 问题2：

### NaN === NaN 为什么是 false？

回答：

> NaN 表示无效数字结果，它不等于任何值，包括自身。如果需要判断 NaN，应使用 Number.isNaN()。

------

## 问题3：

### 如何判断一个值是不是 NaN？

回答：

推荐：

```
Number.isNaN(value)
```

因为它不会进行隐式类型转换。

### 面试会问：模板字符串和普通字符串区别？

回答：

> 模板字符串是 ES6 新增的字符串写法，使用反引号包裹，可以通过 `${}` 插入变量或表达式，并且支持多行字符串，相比传统字符串拼接具有更好的可读性。

## 为什么 typeof null 是 object？

不要回答：

❌ 因为 null 是对象

这是错误的。

正确：

> null 实际上是 JavaScript 的基本数据类型之一，但是由于 JavaScript 早期实现中采用类型标签机制，null 的类型标签和 object 相同，所以 typeof null 返回 object。这是历史遗留问题，为了保持兼容性一直没有修改。

## 1. JavaScript 有哪些数据类型？

标准回答：

> JavaScript 数据类型分为基本数据类型和引用数据类型。基本类型包括 Number、String、Boolean、Undefined、Null、Symbol、BigInt。引用类型主要包括 Object、Array、Function 等。

------

## 2. 基本类型和引用类型区别？

回答：

> 基本类型存储的是实际值，复制时会创建独立的数据；引用类型存储的是对象地址，复制时复制的是引用地址，多个变量可能指向同一个对象。

------

## 3. 为什么 typeof null 是 object？

回答：

> 这是 JavaScript 历史遗留问题，早期实现中 null 被错误标记为 object，为了兼容旧代码一直保留。

------

## 4. 为什么修改对象会影响另一个变量？

例如：

```
let a={
x:1
};

let b=a;

b.x=2;
```

回答：

> 因为对象属于引用类型，a 和 b 保存的是同一个对象地址。

------

## 5. 如何判断数组？

不要：

```
typeof arr
```

因为：

```
object
```

应该：

```
Array.isArray(arr)
```
