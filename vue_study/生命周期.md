# 一、先理解：什么是生命周期？

先不要背钩子名字。

我们先理解一个组件的一生。

例如你的：

```
Home.vue
```

它从出现到消失经历：

```
创建

↓

准备渲染

↓

显示到页面

↓

用户使用

↓

离开页面

↓

销毁
```

这就叫：

> 组件生命周期。

就像一个人：

```
出生

↓

成长

↓

工作

↓

退休

↓

离开
```

Vue组件也一样。

------

# 二、为什么需要生命周期？

思考你的后台首页：

用户登录：

```
/login

↓

/home
```

进入首页。

此时：

你需要请求数据：

例如：

```
axios.get("/user/list")
```

问题：

什么时候发送请求？

你不能：

随便写：

```
<script setup>

axios.get()

</script>
```

虽然能运行，但是：

代码执行时机不明确。

所以Vue提供：

生命周期。

告诉你：

> 组件什么时候创建完成，什么时候显示，什么时候销毁。

------

# 三、Vue3生命周期主要有哪些？

Vue3常用：

| 阶段     | 作用             | 钩子        |
| -------- | ---------------- | ----------- |
| 创建阶段 | 组件初始化       | setup       |
| 挂载阶段 | 页面显示         | onMounted   |
| 更新阶段 | 数据变化导致更新 | onUpdated   |
| 销毁阶段 | 离开页面         | onUnmounted |

实际开发：

你80%的时间用：

```
onMounted()

onUnmounted()
```

# 五、重点：onMounted（最重要）

## 什么叫mounted？

mounted：

翻译：

> 挂载完成

什么意思？

就是：

组件已经出现在页面上。

例如：

你的：

```
<h1>
首页
</h1>
```

已经显示。

这个时候：

执行：

```
onMounted()
```

------

代码：

```
<script setup>


import {onMounted} from "vue"


onMounted(()=>{


console.log("页面加载完成")


})


</script>
```

------

执行顺序：

```
setup执行

↓

template生成

↓

DOM插入页面

↓

onMounted执行
```

# 八、onUnmounted（销毁）

什么时候发生？

比如：

你从：

```
/home
```

跳到：

```
/login
```

Home组件消失。

这时候：

执行：

```
onUnmounted()
```

------

例如：

定时器。

错误：

```
setInterval(()=>{

console.log("刷新")

},1000)
```

问题：

你离开页面：

定时器还在运行。

造成：

内存泄漏。

------

正确：

```
import {
onMounted,
onUnmounted
} from "vue"


let timer



onMounted(()=>{


timer=setInterval(()=>{


console.log("刷新")


},1000)


})



onUnmounted(()=>{


clearInterval(timer)


})
```

------

流程：

进入页面：

```
创建定时器
```

离开：

```
清除定时器
```

# 十一、面试重点问题

## 问题1：

### setup和onMounted执行顺序？

答案：

```
setup

↓

onMounted
```

原因：

setup负责初始化。

mounted表示页面已经挂载完成。

------

## 问题2：

为什么接口请求放onMounted？

标准回答：

> 因为onMounted表示组件已经完成DOM挂载，此时组件已经初始化完成，适合执行依赖页面加载后的操作，例如发送请求、操作DOM等。

------

## 问题3：

onUnmounted有什么作用？

答案：

> 用于组件销毁前清理副作用，例如清除定时器、取消订阅、防止内存泄漏。

# 十三、Vue里面代码大概分四类：

------

# 第一类：初始化逻辑

例如：

页面打开请求数据：

使用：

```
onMounted()
```

例：

```
onMounted(()=>{

getUserList()

})
```

------

# 第二类：用户行为

例如：

点击按钮：

输入内容：

提交表单。

直接写函数。

例如：

```
function submit(){

login()

}
```

------

# 第三类：数据计算

使用：

computed。

例如：

```
const total=computed(()=>{

return list.value.length

})
```

------

# 第四类：数据变化触发动作

使用：

watch。

例如：

搜索：

```
watch(keyword,()=>{

search()

})
```

------

所以：

你的脑中应该形成这个分类：

```
页面打开
    |
    |
 onMounted


用户操作
    |
    |
 function


数据计算
    |
    |
 computed


数据变化产生副作用
    |
    |
 watch


页面离开
    |
    |
 onUnmounted
```
