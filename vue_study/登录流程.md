**：完整登录流程实现**。

这一部分是企业 Vue 项目非常典型的一条链路：

```
Login.vue

↓

api/user.js

↓

utils/request.js

↓

axios

↓

后端接口

↓

返回token

↓

保存token

↓

跳转首页
```

我们今天要把你的登录从：

```
router.push("/home")
```

升级成：

```
输入账号密码

↓

调用登录接口

↓

获取token

↓

保存token

↓

进入后台首页
```

------

# 一、为什么还需要 api/user.js？

现在你的项目：

```
Login.vue
    |
    |
request.js
    |
    |
服务器
```

如果Login直接写：

```
request.post("/login")
```

短期没问题。

但是大型项目：

接口会越来越多：

```
登录接口
用户列表接口
订单接口
权限接口
菜单接口
商品接口
```

如果全部写在页面：

会混乱。

所以企业项目分层：

------

## 页面层（views）

负责：

> 页面交互

例如：

```
用户点击登录
```

------

## api层

负责：

> 管理接口地址

例如：

```
login()

getUserInfo()

getMenu()
```

------

## request层

负责：

> 请求规则

例如：

token

错误处理

baseURL

------

结构：

```
src

├── api
│
│   └── user.js
│
├── utils
│
│   └── request.js
│
├── views
│
│   └── Login.vue
```

# 二、创建 api/user.js

创建：

```
src/api/user.js
```

写：

```
import request from "@/utils/request"


// 登录接口

export function login(data){

    return request.post(
        "/login",
        data
    )

```

------

现在解释每一行。

------

## 第一行

```
import request from "@/utils/request"
```

为什么？

因为：

所有请求走统一封装。

以前：

```
axios.post()
```

现在：

```
request.post()
```

好处：

自动：

- 加token
- 错误处理
- baseURL

------

## login函数

```
export function login(data){

}
```

为什么接收data？

因为登录需要：

用户名

密码

例如：

```
{
 username:"admin",
 password:"123456"
}
```

------

## request.post

```
request.post(
"/login",
data
)
```

等价：

发送：

```
POST /login


{
username:"admin",
password:"123456"
}
```

# 四、完整Login.vue

```
<script setup>


import {

reactive,

computed

}

from "vue"


import {

useRouter

}

from "vue-router"


import {

login

}

from "@/api/user"




const router = useRouter()





// 表单

const form = reactive({

username:"",

password:""

})





// 登录按钮状态

const canLogin = computed(()=>{


return (

form.username &&

form.password

)


})





// 登录

async function handleLogin(){



try{


const res = await login(form)



console.log(
"登录返回:",
res
)



// 保存token

localStorage.setItem(

"token",

res.token

)



// 跳首页

router.push("/home")



}catch(error){


console.log(
"登录失败",
error
)


}



}



</script>
```

# 五、重点理解登录流程

现在点击按钮：

```
<button @click="handleLogin">
登录
</button>
```

执行：

------

## 第一步

进入：

```
handleLogin()
```

------

## 第二步

调用：

```
login(form)
```

注意：

这里不是：

```
axios.post()
```

而是：

```
api/user.js
```

------

## 第三步

进入：

```
user.js
```

执行：

```
request.post("/login",data)
```

------

## 第四步

进入：

```
request.js
```

经过：

请求拦截器。

流程：

```
request.post()

↓

请求拦截器

↓

添加Authorization

↓

发送请求
```

------

## 第五步

服务器返回：

例如：

```
{
    "code":200,
    "token":"abc123"
}
```

------

## 第六步

响应拦截器：

```
return response.data
```

所以：

Login收到：

```
{
 token:"abc123"
}
```

------

## 第七步

保存：

```
localStorage.setItem(
"token",
res.token
)
```

浏览器：

保存：

```
token=abc123
```

------

## 第八步

跳转：

```
router.push("/home")
```

完成登录。

# 七、现在模拟登录返回token

我们暂时修改：

api/user.js

改成：

```
import request from "@/utils/request"



export function login(data){


return new Promise((resolve)=>{


setTimeout(()=>{


resolve({

token:"abc123",

username:data.username


})


},1000)



})


}
```

------

现在发生什么？

点击登录：

```
输入：admin

密码：123456


↓

等待1秒


↓

返回token


↓

保存


↓

进入home
```

------

这里你刚好复习：

Promise。

因为：

```
new Promise()
```

返回：

Promise对象。

所以：

可以：

```
await login(form)
```
